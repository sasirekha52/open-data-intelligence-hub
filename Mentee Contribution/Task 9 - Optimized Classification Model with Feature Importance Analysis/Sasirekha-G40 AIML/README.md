# Predicting E-Commerce Purchase Likelihood

Binary classification project: predict whether an e-commerce visitor will complete a purchase,
using Logistic Regression, Decision Tree, and Random Forest models, with hyperparameter tuning,
feature-importance analysis, and business recommendations.

## Folder structure

```
mini-project-5/
├── data/
│   ├── ecommerce_customer_data.csv     # dataset (generated)
│   └── generate_dataset.py             # script that generated it
├── notebooks/
│   ├── purchase_prediction_analysis.ipynb   # main deliverable
│   └── build_notebook.py               # script that built the notebook (reference only)
├── models/
│   └── purchase_prediction_model.pkl   # saved best pipeline (created after running the notebook)
├── reports/                            # export your feature-importance / recommendations write-ups here
├── presentation/                       # place your slide deck here
├── requirements.txt
└── README.md
```

## How to run (VS Code)

1. Open this folder in VS Code (`File > Open Folder...`).
2. Create and activate a virtual environment (see chat explanation for exact commands).
3. Install dependencies: `pip install -r requirements.txt`.
4. Open `notebooks/purchase_prediction_analysis.ipynb`, select the venv's kernel, and
   **Run All**.
5. The trained model is saved to `models/purchase_prediction_model.pkl`.

## Dataset

`data/ecommerce_customer_data.csv` is a **synthetic** dataset (10,000+ rows) generated by
`data/generate_dataset.py`, since no real dataset was supplied. It includes demographic fields,
behavioural/engagement fields, and a `Purchase` target, plus injected missing values and duplicate
rows to mimic real-world messiness. Swap in a real dataset with the same column names to use your
own data — no other code changes are required.
